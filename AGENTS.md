# Repository Guidelines

## Project Structure & Module Organization
- `src/`: TypeScript source for the CLI/TUI app. Current entrypoint is `src/index.tsx`.
- `dist/`: Build output generated by TypeScript (`tsc`). Do not edit generated files directly.
- `agent/`: Docker environment used to run the project in a consistent container (`agent/Dockerfile`, mounted config/state).
- Root config: `package.json`, `tsconfig.json`, `.editorconfig`, `Makefile`, `pnpm-lock.yaml`.
- `.codex/`: Local Codex skill/config data used by the development workflow.

## Build, Test, and Development Commands
- `make install`: Install dependencies (preferred package manager is pnpm).
- `make start`: Run the app directly from source using `tsx`.
- `make build`: Compile TypeScript to `dist/`.
- `make agent`: Build and start the Dockerized development agent.
- `make attach`: Attach to the running agent container.

## Coding Style & Naming Conventions
- Follow `.editorconfig`: 2-space indentation, LF line endings, UTF-8, trim trailing whitespace (except Markdown), and final newline.
- Use TypeScript + ESM modules (`"type": "module"`).
- Prefer `camelCase` for variables/functions, `PascalCase` for React components, and clear constant names (e.g., `exitSignals`).
- Keep files focused: one primary component/module responsibility per file.

## Testing Guidelines
- No test framework is configured yet.
- For new features, add tests with the framework you introduce and document the command in `Makefile` (for example, `make test`).
- Name tests by behavior, e.g., `src/__tests__/app-exit-signals.test.ts`.
- At minimum, verify startup, signal handling, and clean exit behavior for CLI flows.

## Commit & Pull Request Guidelines
- Current history uses short, imperative commit subjects (examples: `Setup codex in Docker`, `Add ink-tui skill`).
- Keep commit messages concise and action-oriented; scope one logical change per commit.
- PRs should include:
  - A brief problem/solution summary.
  - Linked issue (if applicable).
  - Local verification steps and command output (e.g., `make build`).
  - Screenshots or terminal recordings for UI/TUI behavior changes.
